TOPIC: RECURSION
Category: Data Structures and Algorithms (DSA)

DEFINITION:
Recursion is a programming technique where a function calls itself directly or indirectly to solve a problem by breaking it down into smaller, identical subproblems. Every recursive function must have a base case (the stopping condition) and a recursive case (how the function calls itself with a smaller input). Recursion uses the system call stack to manage function calls.

KEY CONCEPTS:
- Base Case: The condition that stops the recursion (e.g., n == 0, array is empty, node is NULL). Without it, recursion causes infinite loop → Stack Overflow.
- Recursive Case: The part where the function calls itself with a smaller or simpler version of the problem.
- Call Stack: Each recursive call is pushed onto the system stack as a new stack frame with its own local variables. When the base case is reached, frames are popped in reverse order.
- Stack Overflow: Occurs when recursion is too deep and the call stack runs out of memory.
- Direct Recursion: A function calls itself directly.
- Indirect Recursion: Function A calls function B, which calls function A.
- Tail Recursion: The recursive call is the last operation in the function — some compilers optimize this into iteration (tail call optimization), avoiding stack growth.
- Backtracking: A recursive technique that tries all possibilities and undoes (backtracks) choices that don't lead to a solution.
- Time Complexity: Determined by the recurrence relation — solved using the Master Theorem or recursion tree method.

EXPLANATION:
Recursion is powerful because it naturally mirrors the structure of problems that have self-similar substructure — trees, graphs, divide-and-conquer problems, and combinatorial problems. When a recursive function is called, the system pushes a new stack frame onto the call stack containing the function's local variables and return address. When the base case is hit, the stack unwinds — frames are popped and results are returned back up the call chain. The key insight is: if you can solve a problem for a smaller input, and you know how to combine that solution to solve the larger problem, recursion works. Space complexity of recursion is O(depth of recursion), which is O(n) for linear recursion and O(log n) for divide-and-conquer recursion. Backtracking extends recursion by exploring all options and pruning invalid branches — it solves constraint satisfaction problems like N-Queens, Sudoku, and subset generation.

EXAMPLE:
Factorial using recursion:
  factorial(n) = n * factorial(n-1), base case: factorial(0) = 1
  factorial(4):
    4 * factorial(3)
      3 * factorial(2)
        2 * factorial(1)
          1 * factorial(0) = 1
        returns 1 * 1 = 1
      returns 2 * 1 = 2
    returns 3 * 2 = 6
  returns 4 * 6 = 24

Fibonacci (Recursive — Inefficient O(2^n)):
  fib(n) = fib(n-1) + fib(n-2), base: fib(0)=0, fib(1)=1
  fib(5) calls fib(4) and fib(3), fib(4) calls fib(3) and fib(2) — many repeated calls!
  Fix: Memoize results (Dynamic Programming) → O(n)

Backtracking — Generate All Subsets:
  Input: [1, 2, 3]
  At each index, choose to include or exclude the element:
  [] → [1] → [1,2] → [1,2,3], [1,3], [2] → [2,3], [3], []
  All subsets: {}, {1}, {2}, {3}, {1,2}, {1,3}, {2,3}, {1,2,3}

Common Recursive Interview Problems:
- Tower of Hanoi
- Generate all permutations and combinations
- N-Queens problem
- Merge Sort and Quick Sort (divide and conquer)
- Flood Fill (paint bucket)
- Rat in a Maze

INTERVIEW TIP:
When solving recursion problems: (1) identify the base case first, (2) assume the recursive call works correctly for a smaller input, (3) use that result to solve the current level. This "leap of faith" principle simplifies recursive thinking. Always analyze time and space complexity using the recurrence relation.
