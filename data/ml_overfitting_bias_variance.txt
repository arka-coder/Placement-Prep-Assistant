TOPIC: OVERFITTING AND BIAS-VARIANCE TRADEOFF
Category: Machine Learning (ML)

DEFINITION:
Overfitting occurs when a machine learning model learns not just the true underlying patterns in the training data, but also the noise and random fluctuations specific to that dataset. The result is a model that performs excellently on training data but generalizes poorly to new, unseen data. The Bias-Variance Tradeoff is the fundamental tension between two sources of prediction error — bias (error from overly simple assumptions) and variance (error from excessive sensitivity to training data fluctuations) — that determines a model's ability to generalize.

KEY CONCEPTS:
- Overfitting: Training accuracy >> Test accuracy. Model too complex — memorizes training data.
- Underfitting: Both training and test accuracy are low. Model too simple — fails to capture patterns.
- Bias: Systematic error from incorrect model assumptions. High bias = model is too rigid (underfitting).
- Variance: Error from excessive sensitivity to the specific training set. High variance = model changes significantly with different training data (overfitting).
- Total Error: Bias² + Variance + Irreducible Noise (noise in data that cannot be eliminated).
- Irreducible Error: The minimum possible error due to inherent noise in the data. Cannot be reduced by any model.
- Learning Curves: Plots of training and validation error vs training set size or training epochs. Used to diagnose overfitting/underfitting.
- Regularization: Technique to reduce overfitting by adding a penalty for model complexity to the loss function.
  - L1 (Lasso): Adds λΣ|w| penalty → drives some weights to exactly zero → feature selection.
  - L2 (Ridge): Adds λΣw² penalty → shrinks all weights toward zero, none become exactly zero.
  - Elastic Net: Combination of L1 and L2.
- Dropout: Randomly deactivates a fraction of neurons during training in neural networks — prevents co-adaptation and reduces overfitting.
- Early Stopping: Monitor validation loss during training and stop when it starts increasing — prevents over-training.
- Cross-Validation: k-fold evaluation to get robust performance estimates and detect overfitting.
- Data Augmentation: Artificially expand training data by applying transformations (flipping, rotation, noise addition) — reduces overfitting.

EXPLANATION:
The bias-variance tradeoff is fundamental to understanding model behavior. Bias is the error introduced by approximating a complex real-world problem with a simple model — a linear model trying to fit non-linear data has high bias. Variance is the error introduced by a model being too sensitive to the training data — a very deep neural network trained on a small dataset will perform dramatically differently on different samples of the training data. The goal of machine learning is to find the "sweet spot" — a model complex enough to capture the true patterns but not so complex that it memorizes noise. Total expected error = Bias² + Variance + Irreducible Error. Reducing bias (making the model more complex) increases variance, and vice versa. Regularization, ensemble methods, and cross-validation are the main tools to navigate this tradeoff.

EXAMPLE:
Learning Curve Diagnosis:
  Overfitting: Training error = 2%, Validation error = 20%
    → Large gap: model memorizes training data but doesn't generalize.
    Fix: Add regularization, reduce model complexity, get more data, use dropout.

  Underfitting: Training error = 20%, Validation error = 21%
    → Both high: model hasn't learned the patterns.
    Fix: Use more complex model, add more features, reduce regularization, train longer.

  Good Fit: Training error = 5%, Validation error = 7%
    → Small gap, both acceptable: model generalizes well.

Bias-Variance Examples:
  High Bias (Underfitting):
    True function: y = x² + sin(x) (non-linear)
    Model: y = ax + b (linear) → systematically wrong, misses the curve
    Symptom: Both train and test error are high

  High Variance (Overfitting):
    Model: 15th-degree polynomial
    Train error: 0.1% (fits all training points exactly)
    Test error: 35% (wildly wrong on new points)
    Symptom: Large train-test gap

Regularization Effect on Weights:
  Without L2: weights = [12.3, -8.7, 45.1, -32.6] (large, scattered)
  With L2 (λ=0.1): weights = [3.2, -2.1, 8.4, -6.9] (shrunk toward zero, smoother model)

Dropout Example:
  Original layer: [n1, n2, n3, n4, n5, n6]
  With dropout=0.5: randomly deactivate half → [n1, 0, n3, 0, n5, 0]
  Different neurons dropped each batch → model learns redundant representations → robust

Model Complexity vs Error (U-shaped curve):
  Simple model → High Bias, Low Variance (underfitting)
  Optimal model → Low Bias, Low Variance (sweet spot)
  Complex model → Low Bias, High Variance (overfitting)
  Total error is U-shaped — minimum in the middle.

INTERVIEW TIP:
Always use this framework: High Training Error → High Bias (underfitting) → need more complex model. Low Training Error + High Validation Error → High Variance (overfitting) → need regularization, more data, or simpler model. The four key fixes for overfitting: (1) Regularization (L1/L2), (2) More training data, (3) Reduce model complexity, (4) Dropout/Early stopping. Know this cold for any ML interview.
