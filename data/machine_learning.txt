Machine Learning (ML) Study Notes

1. OVERFITTING
Definition: Overfitting occurs when a machine learning model learns not just the underlying patterns in the training data, but also the noise and random fluctuations specific to that dataset. As a result, the model performs very well on training data but generalizes poorly to new, unseen data.
Explanation: The core symptom of overfitting is a large gap between training accuracy (very high) and test/validation accuracy (significantly lower). It happens when a model is too complex relative to the amount of training data — for example, a very deep neural network trained on a small dataset. The model essentially memorizes the training set rather than learning to generalize.
Causes: Model is too complex (too many parameters), insufficient training data, training for too many epochs, no regularization applied, features contain noise or irrelevant attributes.
Solutions: Use a simpler model (reduce parameters/layers), increase training data size, apply Early Stopping during training, use Cross-Validation for reliable evaluation, apply Regularization (L1/Lasso or L2/Ridge), use Dropout in neural networks, perform Data Augmentation to synthetically expand the training set.
Interview Tip: Overfitting is best detected by plotting learning curves (training vs validation loss over epochs). A model that overfits will show training loss still decreasing while validation loss starts increasing — this is the point to stop training (early stopping).

2. UNDERFITTING
Definition: Underfitting occurs when a machine learning model is too simple to capture the underlying patterns in the training data. Unlike overfitting, an underfitting model performs poorly on both training and test data.
Explanation: Underfitting results in high bias — the model makes oversimplified assumptions about the data. For example, using a linear regression model to capture a highly non-linear relationship. Both training and test errors are high, indicating the model has not learned the data's structure.
Causes: Model too simple (too few parameters), insufficient or irrelevant features provided, too much regularization penalizing the model, too few training epochs (model hasn't converged).
Solutions: Use a more complex model with more parameters/layers, add more relevant and informative features, reduce regularization strength, train for more epochs to allow convergence, use ensemble methods to increase model capacity.
Interview Tip: Underfitting = High Bias + Low Variance. The model is too rigid. The fix is always to increase model capacity or improve features — the opposite of overfitting fixes.

3. BIAS-VARIANCE TRADEOFF
Definition: The Bias-Variance Tradeoff is a fundamental concept in machine learning that describes the tension between two sources of error that affect model performance: Bias (error from overly simplistic assumptions) and Variance (error from excessive sensitivity to training data fluctuations).
Explanation: Bias is the difference between the model's average prediction and the true value — high bias means the model is systematically wrong (underfitting). Variance measures how much the model's predictions change across different training sets — high variance means the model is overly sensitive to training data (overfitting). The total expected prediction error decomposes as: Total Error = Bias² + Variance + Irreducible Error (noise). Reducing bias typically increases variance and vice versa — this is the tradeoff. The goal is to find the "sweet spot" — a model complex enough to capture patterns but not so complex that it memorizes noise.
Interview Tip: Simple models (linear regression) → High Bias, Low Variance. Complex models (deep neural networks) → Low Bias, High Variance. Regularization, cross-validation, and ensemble methods are tools to balance this tradeoff. Always mention this tradeoff when discussing model selection.

4. REGULARIZATION
Definition: Regularization is a set of techniques used to prevent overfitting in machine learning models by adding a penalty term to the loss function that discourages model complexity (large weights). It forces the model to stay simple while still fitting the training data well.
Explanation: L1 Regularization (Lasso) adds the penalty λ × Σ|weights| to the loss. It encourages sparse models where some weights become exactly zero — effectively performing automatic feature selection. It is used in linear/logistic regression. L2 Regularization (Ridge) adds the penalty λ × Σ(weights²). It shrinks all weights toward zero but does not eliminate them completely — better for handling multicollinearity. Elastic Net combines L1 and L2 penalties, balancing feature selection (L1) and weight shrinkage (L2). The hyperparameter λ (lambda) controls regularization strength — larger λ = stronger penalty = simpler model.
Interview Tip: Key difference: L1 produces sparse models (feature selection), L2 produces dense models (all features kept but shrunk). If you suspect many irrelevant features, use L1 (Lasso). If features are correlated, use L2 (Ridge). Elastic Net is the safe default when unsure.

5. CROSS-VALIDATION
Definition: Cross-Validation (CV) is a model evaluation technique that tests how well a model generalizes to an independent dataset. Instead of using a single train/test split, it divides data into multiple subsets and trains/tests the model multiple times to get a more robust performance estimate.
Explanation: K-Fold Cross-Validation divides data into K equal folds. In each of K iterations, one fold is used as the test set and the remaining K-1 folds are used for training. Final performance is the average across all K iterations. Common choices: K=5 or K=10. Stratified K-Fold maintains the class distribution in each fold — essential for imbalanced datasets. Leave-One-Out Cross-Validation (LOOCV) sets K = number of samples — very accurate but computationally expensive. Time Series Cross-Validation respects temporal order (no future data leaking into training).
Interview Tip: Cross-validation solves the problem of train/test split being "lucky" or "unlucky". It provides a more reliable estimate of generalization error. Always use stratified K-fold for classification tasks to maintain class ratios in each fold.

6. MODEL EVALUATION METRICS
Definition: Model Evaluation Metrics are quantitative measures used to assess how well a machine learning model performs on a given task. Choosing the right metric depends on the problem type (classification vs regression) and the business objective.
Explanation for Classification: Accuracy = (TP + TN) / Total — suitable only for balanced datasets. Precision = TP / (TP + FP) — how many predicted positives are truly positive; important when false positives are costly (e.g., spam detection). Recall (Sensitivity) = TP / (TP + FN) — how many actual positives are correctly identified; critical when false negatives are costly (e.g., cancer detection). F1-Score = 2 × (Precision × Recall) / (Precision + Recall) — harmonic mean of precision and recall; best for imbalanced datasets. ROC-AUC measures the area under the ROC curve — closer to 1 is better; robust to class imbalance.
Explanation for Regression: MSE (Mean Squared Error) = average of squared differences between predicted and actual values. RMSE = √MSE, interpretable in original units. MAE (Mean Absolute Error) = average of absolute differences, more robust to outliers than MSE.
Interview Tip: Never default to accuracy for imbalanced datasets (e.g., 99% accuracy in fraud detection is meaningless if only 1% of transactions are fraud). Use F1-Score or ROC-AUC. In regression, prefer RMSE when large errors are especially undesirable.

7. FEATURE ENGINEERING
Definition: Feature Engineering is the process of using domain knowledge to select, transform, create, or extract features (input variables) from raw data to improve machine learning model performance. It is often more impactful than choosing a more complex algorithm.
Explanation: Feature Scaling ensures features are on a comparable scale. Normalization scales values to [0, 1] using (x − min) / (max − min) — suitable when the distribution is not Gaussian. Standardization scales to mean=0 and std=1 using (x − mean) / std — preferred for algorithms sensitive to feature magnitudes (SVM, PCA). Feature Selection removes irrelevant or redundant features using Filter methods (correlation, chi-square), Wrapper methods (forward/backward selection), or Embedded methods (L1 regularization during training). Feature Creation involves creating new meaningful features by combining or transforming existing ones — e.g., creating "age_group" from raw age, or polynomial features for non-linear relationships.
Interview Tip: "Garbage in, garbage out" — a poorly engineered feature set cripples even the most sophisticated model. Always perform exploratory data analysis (EDA) first to understand feature distributions, correlations, and missing values before engineering features.

8. ENSEMBLE METHODS
Definition: Ensemble Methods are machine learning techniques that combine predictions from multiple individual models (called base learners or weak learners) to produce a stronger, more accurate final prediction than any single model could achieve alone.
Explanation: Bagging (Bootstrap Aggregating) trains multiple models independently on different random samples (with replacement) of training data and combines predictions by averaging (regression) or majority vote (classification). Key example: Random Forest, which combines many decision trees trained on random subsets of data and features. Bagging primarily reduces variance. Boosting trains models sequentially — each new model focuses on correcting the errors of the previous model by up-weighting misclassified examples. Key examples: AdaBoost, Gradient Boosting, XGBoost, LightGBM. Boosting reduces both bias and variance. Stacking trains multiple diverse base models and uses a meta-learner (another model) trained on their predictions to make the final decision.
Interview Tip: Random Forest = Bagging + Decision Trees. XGBoost = Gradient Boosting with regularization. In competitions, boosting (especially XGBoost/LightGBM) tends to outperform bagging. Mention that ensemble methods trade interpretability for accuracy.

9. SUPERVISED vs UNSUPERVISED LEARNING
Definition: Supervised Learning is a type of machine learning where the model is trained on labeled data — each training example has an input (features) and a known output (label). The goal is to learn a mapping from inputs to outputs. Unsupervised Learning trains on unlabeled data — the model must discover hidden patterns or structure on its own, without pre-specified outputs.
Explanation: Supervised Learning tasks: Classification (predict a discrete label — spam/not spam, cat/dog) and Regression (predict a continuous value — house price, temperature). Key algorithms: Linear Regression, Logistic Regression, Decision Trees, SVM, Neural Networks, k-NN. Unsupervised Learning tasks: Clustering (group similar data points — K-Means, DBSCAN, Hierarchical Clustering), Dimensionality Reduction (reduce feature space — PCA, t-SNE, Autoencoders), and Anomaly Detection. Semi-supervised learning uses a small amount of labeled data and a large amount of unlabeled data.
Interview Tip: A common interview question is "how would you know which to use?" — use supervised learning when you have labeled data and a specific target to predict; use unsupervised learning for exploration, pattern discovery, or when labeling is expensive.

10. NEURAL NETWORKS AND DEEP LEARNING
Definition: An Artificial Neural Network (ANN) is a computational model inspired by the biological neural networks in the brain. It consists of layers of interconnected nodes (neurons) that process information through weighted connections, non-linear activation functions, and a learning process called backpropagation.
Explanation: A neural network has an Input Layer (receives raw features), one or more Hidden Layers (learn intermediate representations), and an Output Layer (produces final prediction). Each neuron computes a weighted sum of inputs, adds a bias, and passes the result through an activation function (ReLU, Sigmoid, Tanh). Training uses Backpropagation — compute loss, then calculate gradients using the chain rule, and update weights using Gradient Descent. Deep Learning refers to neural networks with many hidden layers — capable of learning very complex hierarchical representations. Common architectures: CNN (Convolutional Neural Networks) for images, RNN/LSTM for sequential data, Transformers for NLP.
Interview Tip: ReLU (Rectified Linear Unit) is the most popular activation function in hidden layers because it avoids the vanishing gradient problem and is computationally simple. Sigmoid is used only in the output layer for binary classification. Softmax is used for multi-class classification outputs.

11. LINEAR REGRESSION
Definition: Linear Regression is a supervised learning algorithm that models the relationship between a continuous dependent variable (target) and one or more independent variables (features) by fitting a straight line (or hyperplane in multiple dimensions) to the data.
Explanation: Simple Linear Regression: y = mx + b, where m is the slope (coefficient) and b is the intercept. Multiple Linear Regression: y = w₁x₁ + w₂x₂ + ... + wₙxₙ + b. The model minimizes the Sum of Squared Errors (SSE) between predictions and actual values using Ordinary Least Squares (OLS) or Gradient Descent. Key assumptions: Linearity (relationship between X and y is linear), Independence of errors, Homoscedasticity (constant variance of errors), and Normality of residuals.
Interview Tip: R² (R-squared) measures how much variance in the target is explained by the features — ranges from 0 to 1 (1 = perfect fit). Adjusted R² penalizes for adding irrelevant features. Always check residual plots to validate linear regression assumptions.

12. LOGISTIC REGRESSION
Definition: Logistic Regression is a supervised classification algorithm that predicts the probability of a binary outcome (0 or 1) by applying the sigmoid function to a linear combination of input features. Despite its name, it is a classification algorithm, not a regression algorithm.
Explanation: The model computes a linear combination z = w₁x₁ + w₂x₂ + ... + b, then applies the Sigmoid function σ(z) = 1 / (1 + e^(-z)) to squash the output between 0 and 1, interpreting it as a probability. A threshold (usually 0.5) is applied — if P > 0.5, predict class 1, else class 0. The model is trained using Maximum Likelihood Estimation (MLE) or equivalently minimizing Binary Cross-Entropy loss using Gradient Descent. For multi-class classification, use Softmax Regression (Multinomial Logistic Regression).
Interview Tip: Logistic regression outputs probabilities, not hard predictions — this makes it interpretable. The coefficient of a feature represents the change in log-odds per unit increase in that feature. Mention that Logistic Regression is a linear classifier — it only works well when classes are linearly separable.
