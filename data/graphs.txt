TOPIC: GRAPHS
Category: Data Structures and Algorithms (DSA)

DEFINITION:
A Graph is a non-linear data structure consisting of a set of vertices (nodes) and edges (connections between nodes). Formally, G = (V, E). Unlike trees, graphs can have cycles, multiple paths between nodes, and disconnected components. Graphs model real-world networks: social networks, road maps, web pages, and dependency systems.

KEY CONCEPTS:
- Directed vs Undirected: In directed graphs, edges have a direction (A → B). In undirected, edges are bidirectional (A — B).
- Weighted vs Unweighted: Weighted graphs assign a cost/distance to each edge (e.g., road distances). Unweighted edges have equal cost.
- Cyclic vs Acyclic: Cyclic graphs contain cycles. DAG (Directed Acyclic Graph) has no cycles — used in topological sorting.
- Adjacency Matrix: 2D array where matrix[i][j] = 1 if edge exists. Space: O(V²). Edge lookup: O(1). Best for dense graphs.
- Adjacency List: Array of lists where list[i] contains neighbors of vertex i. Space: O(V+E). Best for sparse graphs.
- BFS (Breadth-First Search): Uses a queue, explores level by level. Finds shortest path in unweighted graphs. Time: O(V+E).
- DFS (Depth-First Search): Uses a stack/recursion, explores as deep as possible before backtracking. Time: O(V+E).
- Topological Sort: Linear ordering of vertices in a DAG such that for every edge u→v, u comes before v. Used in task scheduling.
- Dijkstra's Algorithm: Finds shortest path in weighted graphs with non-negative edges. Uses a min-heap. Time: O((V+E) log V).
- Bellman-Ford: Handles negative weight edges. Time: O(V*E). Detects negative weight cycles.
- Floyd-Warshall: All-pairs shortest path. Time: O(V³).
- Union-Find (Disjoint Set): Detects cycles in undirected graphs and finds connected components. Used in Kruskal's MST.

EXPLANATION:
Graphs are the most versatile data structure for modeling networks and relationships. BFS is the go-to algorithm for shortest path in unweighted graphs because it processes nodes level by level — the first time a node is reached, it is via the shortest path. DFS is better for cycle detection, topological sorting, and finding strongly connected components (Tarjan's, Kosaraju's algorithm). For weighted shortest paths, Dijkstra's is optimal for non-negative weights, but fails with negative weights — use Bellman-Ford instead. Topological sort (Kahn's BFS-based algorithm or DFS-based) is essential for dependency resolution — compilers, build systems, and course prerequisite ordering all use it.

EXAMPLE:
Graph (Undirected):
  Vertices: {0, 1, 2, 3, 4}
  Edges: {(0,1), (0,2), (1,3), (2,4)}
  Adjacency List:
    0: [1, 2]
    1: [0, 3]
    2: [0, 4]
    3: [1]
    4: [2]

BFS from vertex 0:
  Queue: [0], Visited: {0}
  Dequeue 0, enqueue 1, 2 → Queue: [1, 2]
  Dequeue 1, enqueue 3 → Queue: [2, 3]
  Dequeue 2, enqueue 4 → Queue: [3, 4]
  Dequeue 3 (no unvisited neighbors) → Queue: [4]
  Dequeue 4 (no unvisited neighbors) → Queue: []
  BFS Order: 0, 1, 2, 3, 4

Topological Sort (DAG: "Course Prerequisites"):
  Courses: A→C, B→C, C→D
  In-degree: A=0, B=0, C=2, D=1
  Enqueue nodes with in-degree 0: [A, B]
  Process A → reduce C's in-degree to 1 → Process B → reduce C's in-degree to 0 → enqueue C
  Process C → reduce D's in-degree to 0 → enqueue D
  Result: A, B, C, D (valid order to take courses)

Common Interview Problems:
- Number of islands (DFS/BFS on grid)
- Detect cycle in directed/undirected graph
- Topological sort (course schedule problem)
- Shortest path in unweighted graph (BFS)
- Dijkstra's shortest path (weighted graph)
- Clone a graph
- Find number of connected components

INTERVIEW TIP:
Always clarify: directed or undirected? Weighted or unweighted? Cyclic? The answers determine which algorithm to use. For grid problems (islands, maze), treat each cell as a graph vertex and its 4 neighbors as edges — apply BFS or DFS. Use visited set/array to avoid revisiting nodes.
